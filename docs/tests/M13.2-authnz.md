## ŌÜĀ’ĖÅ Edge Cases

| #   | Step | Expected |
|-----|------|----------|
| E1  | Decode `accessToken` and verify the `exp` claim is exactly `iat + 900` | Holds across 3 fresh logins (rules out clock-skew drift) |
| E2  | Wait until 30 s before `exp`, then call a protected GET; on 401 the UI auto-refreshes | One `POST /api/auth/refresh` call observed in DevTools, retry succeeds, no user-visible failure |
| E3  | Cross-tenant: log in as `alt-admin`, request a project that belongs to tenant `0000ŌĆ”0000` | `403 Forbidden`; `RbacEnforcer` blocks before controller body runs; no rows leaked in response |
| E4  | Concurrent refresh: fire 5 `/api/auth/refresh` calls in parallel using the same `am_rt` | Exactly 1 succeeds with a new pair; the other 4 return `401` (refresh-token reuse blocked by the Redis-backed rotation log) |
| E5  | Clock skew: present a token whose `iat` is **5 s in the future** (laptop clock drift) | `200` (Spring's default `JwtTimestampValidator` clock skew is ┬▒60 s) |
| E6  | Viewer reads workflow + blackboard | `viewer` `GET /api/workflows` ŌåÆ `200`; `viewer` `GET /api/blackboard/entries` ŌåÆ `200` (read-only roles) |
| E7  | Cookie-only session (no `localStorage`) ŌĆö open `/dashboard/orchestration` directly after a fresh login from the SAME browser | `middleware.ts` sees `am_at` cookie ŌåÆ renders without bounce; first fetch reads token from the mirror cookie via `token-store.ts` |

## ŌØī Failure Modes

| #   | Step | Expected |
|-----|------|----------|
| F1  | `GET /api/workflows` with **no** `Authorization` header (`AUTH_ENFORCED=true`) | `401`; `WWW-Authenticate: Bearer realm="agentmesh"` |
| F2  | `GET /api/workflows` with a token whose signature is tampered (last byte mutated) | `401`; Spring logs `JwtException: signature mismatch` at `WARN` |
| F3  | `viewer` calls `POST /api/workflows/start` | `403`; `@PreAuthorize` denies; controller method body **not** entered (`@Transactional` not opened) |
| F4  | `POST /api/auth/logout` with valid pair, then re-use the **old** access token | `401` (jti is in the Redis blocklist); refresh token also rejected |
| F5  | Open WS without `?token=`: `wscat -c ws://api.agentmesh.localhost/ws` | Server closes with code **4401**, reason `unauthenticated`; UI receives `onclose` and triggers the `refreshAccessToken()` branch |
| F6  | Open WS with an **expired** token | Same as F5 ŌĆö close 4401, UI refresh-and-reconnect succeeds within Ōēż 1 s |
| F7  | `POST /api/auth/login` with bad credentials 5├Ś in 60 s | All `401 {"message":"Invalid credentials"}`; bucket-rate-limit middleware (`agentmesh-ratelimit`) eventually shaves the 6th attempt |
| F8  | `POST /api/auth/refresh` with a token revoked via `/api/auth/logout` | `401`; both cookies wiped on the response (defence in depth) |

## ­¤ż¢ Automated checks (k6)

See [`load-tests/auth-smoke.js`](../../load-tests/auth-smoke.js).

The k6 smoke profile asserts the **acceptance gate**:

> *k6 smoke profile passes with Authorization Bearer header; 401 without.*

Thresholds (fail the build if breached):

```
http_req_failed{auth:bearer}  rate==0
http_req_failed{auth:none}    rate==1   (every request must 401)
http_req_failed{auth:tampered} rate==1
http_req_duration{auth:bearer} p(95)<500
```

Run locally:

```sh
k6 run \
  -e BASE_URL=http://api.agentmesh.localhost \
  -e USERNAME=admin \
  -e PASSWORD=admin-change-me \
  load-tests/auth-smoke.js
```

CI gating: the same script runs in the GitHub Actions `auth-smoke`
job after the stack is brought up via `make staging-foundation-up`.
